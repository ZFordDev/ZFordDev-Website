Core Principles

Frameworks change, but good engineering remains. I build software that respects the user, the hardware, and the next developer who touches the code.

Ship to Finish

I prioritize polish and documentation. Every build should be a reference point for quality, ensuring projects are readable and ready for production.

Resource Respect

Software should be efficient. Whether it's local-first tools or cloud infrastructure, I build with a focus on speed, privacy, and low-overhead.

Composable Logic

I prefer small, decoupled modules over rigid monoliths. Building with interchangeable parts makes systems easier to test, scale, and maintain.
